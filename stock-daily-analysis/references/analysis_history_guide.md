# 📜 历史分析连续性指南

## 🎯 核心目标

确保股票分析的**连续性**和**一致性**，通过历史预测对比提供回测数据，追踪观点演进，避免分析矛盾。

---

## 💡 为什么需要历史分析连续性

### 问题场景
- ❌ 今天说"看多"，明天说"看空"，没有连贯逻辑
- ❌ 上次预测"目标价 60 元"，这次完全忽略
- ❌ 建议演进不清晰：昨天说"持有"，今天说"加仓"，没有理由
- ❌ 风险提示遗忘：上次提到的风险这次完全不提

### 解决方案
- ✅ 读取历史分析，确保观点连贯
- ✅ 对比历史预测与实际走势，评估准确性
- ✅ 追踪观点演进，说明变化原因
- ✅ 持续关注已识别的风险

---

## 🎯 执行步骤

### 步骤 1: 读取历史分析记录

通过读取历史分析报告，获取之前的预测、建议和关键观点，用于连续性分析和预测准确性评估。

```bash
ptrade analysis 股票名称 --action read --count 3
```

参数说明：
- `--action read`: 读取操作
- `--count 5`: 读取最近 5 次分析报告（可根据需要调整，建议 3-7 份）
- 如该股票从未分析过，命令会返回"暂无历史分析记录"，此时跳过连续性分析步骤

### 步骤 2: 分析预测准确性

对比历史预测与实际走势：
- ✅ 预测方向准确性（看多/看空）
- ✅ 目标价位达成率
- ✅ 建议操作的有效性
- ✅ 关键观点验证

### 步骤 3: 识别观点演进

追踪历史分析的变化趋势：
- **观点变化**: 看空 → 看多 → 看空
- **建议演进**: 观望 → 持有 → 加仓 → 减仓
- **风险追踪**: 持续关注的风险点

### 步骤 4: 生成连续性分析报告

```markdown
# {股票名称} - 历史分析连续性报告

**分析日期**: {YYYY-MM-DD HH:mm}
<!-- 注意：必须使用真实的系统时间，可通过 `date` 命令获取，不要使用项目配置中的 currentDate -->

---

## 📊 近期分析对比表
| 日期 | 预测方向 | 目标价位 | 建议操作 | 关键观点 | 实际表现 |
|------|----------|----------|----------|----------|----------|
| 2026-03-25 | 看多 | 58.0 | 持有 | 突破关键阻力位 | ✅ 上涨至 56.5 |
| 2026-03-24 | 谨慎看多 | 55.0 | 观察 | 量能不足，观望 | ✅ 横盘震荡 |
| 2026-03-23 | 看多 | 54.0 | 加仓 | 业绩超预期 | ❌ 未达预期 |

## 📈 观点演进分析
- **预测准确性**: 3次中2次准确（66.7%）
- **观点变化**: 从谨慎看多转为积极看多
- **建议演进**: 观察 → 持有（建议逐步加仓）
- **风险追踪**: 持续关注业绩不及预期风险、量能不足问题

## 💡 连续性判断
上次预测目标价 58.0，当前价 56.5，距离目标还有 2.7% 空间。
近期预测准确率较高（66.7%），建议延续"看多"观点，但需强调量能配合验证。
```

### 步骤 5: 保存到中间数据目录

请参考下列命令将分析报告保存。

💾 保存方式：使用 ptrade CLI 工具的临时数据存储功能
📂 保存位置：ptrade 工作的 workspace/temp-data 目录（通过 STOCK_ANALYSIS_WORKSPACE 环境变量配置）

```bash
# 1. 先将连续性分析报告保存到临时文件
cat > /tmp/{股票名称}_history_continuity_${CLAUDE_SESSION_ID}.md << 'EOF'
{连续性分析报告内容}
EOF

# 2. 使用 --file 参数归档
ptrade temp-data "股票名称" \
  --action save \
  --category history-continuity \
  --file /tmp/{股票名称}_history_continuity_${CLAUDE_SESSION_ID}.md
```

⚠️ 严格禁止使用 write 或 Edit 工具直接操作文件，必须通过 ptrade temp-data 命令保存数据
**重要**: 保存成功后，其他 agent 会自动读取和合并这份报告。

---

## 📋 注意事项

### 数据目录说明

- **历史分析记录**: 保存在 `stocks_analysis/` 目录（股票名称作为目录）
- **中间数据**: 保存在 `temp-data/` 目录（股票名称作为目录，通过 ptrade 的工作空间配置）
- **分析文件命名**: `<股票名>-YYYY-MM-DD-HHMM.md`
- **首次分析**: 如从未分析过该股票，返回"暂无历史分析记录"

### 环境变量配置

确保已配置 `STOCK_ANALYSIS_WORKSPACE` 环境变量（项目级配置）：

```json
{
  "env": {
    "STOCK_ANALYSIS_WORKSPACE": "/path/to/workspace-stock_agent"
  }
}
```

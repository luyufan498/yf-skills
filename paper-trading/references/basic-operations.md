## 初始化资金池

为指定股票创建独立的资金池，每只股票的资金池相互独立。

### 命令语法

```bash
ptrade init "股票名称" --capital 资金金额 [--code 股票代码] [--force]
```

### 参数说明

| 参数 | 必需 | 说明 | 示例 |
|------|------|------|------|
| `股票名称` | 是 | 股票的中文名称 | "赛力斯", "中科曙光" |
| `--capital` / `-c` | 是 | 初始资金金额（元） | `--capital 100000` |
| `--code` | 否 | 股票代码 | `--code sh600000` |
| `--force` / `-f` | 否 | 强制重新初始化 | `--force` |

### 使用示例

```bash
# 基本初始化
ptrade init "赛力斯" --capital 50000

# 指定股票代码初始化
ptrade init "浦发银行" --capital 100000 --code sh600000

# 强制重新初始化（覆盖现有数据）
ptrade init "赛力斯" --capital 100000 --force
```

### 工作原理

1. **创建资金池**：为股票分配指定金额的虚拟资金
2. **自动查询代码**：如未指定 `--code`，系统会自动查询股票代码
3. **数据存储**：在 `intermediate/<股票名>/模拟买卖/` 目录下创建数据文件
4. **重复检查**：如股票已存在，会提示使用 `--force` 重新初始化

### 常见问题

**Q: 为什么需要初始化资金池？**
A: 每只股票需要独立的资金池来追踪资金使用、持仓成本和收益，模拟真实交易的资金管理。

**Q: 初始化后能更改资金金额吗？**
A: 可以，使用 `--force` 选项重新初始化，但这会清空所有交易记录。

**Q: 支持哪些市场？**
A: 支持 A股（sh/sz）、港股（hk）、美股（gb_ 或直接代码）。

## 买入股票

在资金池充足的情况下买入指定数量的股票，系统自动获取实时价格。

### 命令语法

```bash
ptrade buy "股票名称" (--qty 数量 | --amount 金额) [--note 备注]
```

### 参数说明

| 参数 | 必需 | 说明 | 示例 |
|------|------|------|------|
| `股票名称` | 是 | 已初始化的股票名称 | "赛力斯" |
| `--qty` / `-q` | 二选一 | 买入股数 | `--qty 100` |
| `--amount` / `-a` | 二选一 | 买入金额（元） | `--amount 5000` |
| `--note` / `-n` | 否 | 交易备注 | `--note "首次建仓"` |

### 使用示例

```bash
# 按股数买入
ptrade buy "赛力斯" --qty 100

# 按金额买入（系统自动计算股数）
ptrade buy "赛力斯" --amount 5000

# 带备注买入
ptrade buy "赛力斯" --qty 200 --note "初始建仓"

# 按金额买入带备注
ptrade buy "赛力斯" --amount 10000 --note "加仓"
```

### 工作原理

1. **验证资金池**：检查资金池是否已初始化
2. **检查可用资金**：确保有足够资金完成交易
3. **获取实时价格**：自动调用价格查询接口获取最新股价
4. **计算交易金额**：按股数或金额计算总交易额
5. **执行买入**：记录买入价格、数量、金额、时间
6. **更新持仓**：增加持仓数量，更新成本（先进先出原则）
7. **扣减资金**：从可用资金中扣减交易金额

### 注意事项

- **按股数买入**：`--qty` 指定具体股数，交易金额 = 股数 × 实时价格
- **按金额买入**：`--amount` 指定总金额，系统自动计算能购买的股数（向下取整）
- **零股处理**：按金额买入时，零股会被舍去不购买
- **价格波动**：每次交易价格不同，以实际成交时为准
- **资金不足**：如资金不足，交易会失败并提示可用资金余额

### 错误处理

| 错误信息 | 原因 | 解决方案 |
|---------|------|---------|
| "资金池未初始化" | 股票未初始化 | 先运行 `ptrade init` 初始化资金池 |
| "可用资金不足" | 资金不够 | 减少股数/金额，或使用 `--amount` 优先级 |
| "无法获取实时价格" | 网络或API问题 | 检查网络，稍后重试 |

## 卖出股票

卖出持有的股票，支持部分卖出和全仓卖出，系统自动计算盈亏。

### 命令语法

```bash
ptrade sell "股票名称" (--qty 数量 | --all) [--note 备注]
```

### 参数说明

| 参数 | 必需 | 说明 | 示例 |
|------|------|------|------|
| `股票名称` | 是 | 已初始化的股票名称 | "赛力斯" |
| `--qty` / `-q` | 二选一 | 卖出股数 | `--qty 50` |
| `--all` / `-a` | 二选一 | 全部卖出 | `--all` |
| `--note` / `-n` | 否 | 交易备注 | `--note "部分止盈"` |

### 使用示例

```bash
# 按股数卖出
ptrade sell "赛力斯" --qty 50

# 全仓卖出
ptrade sell "赛力斯" --all

# 带备注卖出
ptrade sell "赛力斯" --qty 100 --note "部分止盈"

# 全仓卖出带备注
ptrade sell "赛力斯" --all --note "清仓离场"
```

### 工作原理

1. **验证持仓**：检查持仓数量是否足够
2. **获取实时价格**：自动调用价格查询接口获取最新股价
3. **计算成本**：根据先进先出（FIFO）原则计算卖出成本
4. **计算盈亏**：交易金额 - 成本 = 盈亏（正数为盈，负数为亏）
5. **执行卖出**：记录卖出价格、数量、金额、成本、盈亏、时间
6. **更新持仓**：减少持仓数量
7. **增加资金**：回收交易金额到可用资金池

### 成本计算（FIFO）

先进先出原则示例：

```
买入记录：
1. 第1批买入 100 股，成本价 10 元/股
2. 第2批买入 100 股，成本价 12 元/股
3. 第3批买入 100 股，成本价 13 元/股

卖出操作：
- 卖出 50 股 → 成本来自第1批（10 元/股）
- 再卖出 50 股 → 成本来自第1批剩余部分（10 元/股）
- 再卖出 100 股 → 成本来自第2批（12 元/股）
```

### 盈亏计算

```python
盈亏 = 卖出金额 - 卖出成本
收益率 = (盈亏 / 卖出成本) × 100%
```

### 注意事项

- **持仓验证**：卖出数量不能超过持仓数量
- **成本基础**：每批买入价格可能不同，影响单笔交易盈亏
- **浮动盈亏**：持仓股票的当前市值与成本价差额
- **实现盈亏**：已卖出部分的盈亏（已锁定）

### 错误处理

| 错误信息 | 原因 | 解决方案 |
|---------|------|---------|
| "持仓数量不足" | 卖出数 > 持仓数 | 减少卖出数或使用 `--all` |
| "无持仓可卖" | 持仓为0 | 先买入股票 |
| "无法获取实时价格" | 网络或API问题 | 检查网络，稍后重试 |

## 强制操作

### 强制重新初始化

使用 `--force` 选项可以覆盖现有的交易数据：

```bash
ptrade init "股票名称" --capital 新金额 --force
```

**风险提示**：这将永久删除所有历史交易记录和持仓数据！

### 强制删除账户

使用 `--force` 选项可以删除有持仓的账户：

```bash
ptrade delete "股票名称" --force
```

**风险提示**：这将永久删除账户和所有相关数据！

## 注意事项

### 交易前检查清单

在执行交易前，请确认：

1. ✅ 已初始化资金池（`ptrade init`）
2. ✅ 资金池有足够可用资金（`ptrade pool`）
3. ✅ 股票代码正确（自动查询或手动指定）
4. ✅ 网络连接正常（能获取实时价格）
5. ✅ 明确交易数量或金额
6. ✅ 想好交易备注（用于后续追踪）

### 数据实时保存

- 所有交易操作都会立即保存到 `operations.json` 文件
- 持仓状态更新保存到 `account.json` 文件
- 无需手动保存，系统自动处理

### 价格自动获取

- 每次 `buy`/`sell` 操作都会自动获取最新价格
- 价格来源：腾讯财经（A股/港股）、新浪财经（美股）
- 价格为实时或准实时，可能有轻微延迟

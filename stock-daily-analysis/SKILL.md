# 股票深度分析

执行系统化的股票深度分析流程，通过自适应搜索和多数据源整合，生成专业的股票分析报告。

## 核心流程

分析流程采用并行执行策略，通过多个专用 subagent 同时处理不同维度任务：

1. **获取股价数据** - 实时股价、涨跌幅、K 线趋势 + 标准化股票信息
2. **读取历史分析** - 建立分析连续性和趋势对比
3. **自适应深度搜索** - 多轮迭代收集新闻和热点
4. **获取专业数据** - 龙虎榜、财务基本面、ETF 榜单
5. **生成分析报告** - 三层分析框架
6. **保存报告** - 结构化存储

**执行特点**：
- ⚡ **并行执行**：历史、搜索、广发数据三个 subagent 并行工作
- 🔗 **统一管理**：所有 subagent 由主 session 创建，确保完全隔离
- 📊 **多维整合**：历史、新闻、专业数据、股价全面融合


## 执行步骤

### 步骤 1: 获取股价数据

**主 agent 直接执行**：调用 stock-market-data skill 获取实时股价、K 线数据以及标准化的股票名称和代码。

**执行原则**（主 agent 操作）：
- 主 agent 直接调用 stock-market-data skill
- 用户输入的名称可能不准确，进行模糊匹配和验证，确保获取到正确的股票名称和代码，供后续步骤使用
- 获取标准化股票信息后，将其保存为全局变量或传递给后续步骤使用

**执行步骤**（主 agent 直接操作）：
1. 根据用户输入（股票名称/代码/简称），调用 stock-market-data skill 获取以下数据：
   - **股票信息**：
     - 完整的股票名称（中文全称）
     - 正确的股票代码（如：600519.SH、000858.SZ）
     - 所属交易所信息（上交所/深交所）
   - **实时股价**：当前价、昨收、涨跌幅、成交量
   - **K 线数据**：30 日日线趋势
2. 如数据获取失败，标注"数据暂不可用，建议使用东方财富/同花顺查看"
3. 使用标准化股票名称(简体中文)和代码继续后续步骤

### 步骤 2: 读取历史分析

创建 subagent，要求 subagent 调用 analysis_manager 工具读取历史分析报告并生成连续性分析。

**执行原则**（主 agent 操作）：
- 创建专门的 subagent
- 在 subagent 的 prompt 中明确要求其读取历史分析数据并保存连续性报告

**任务要求**（传递给 subagent 的 prompt 内容）：
1. subagent 读取 `references/analysis_history_guide.md` 了解执行方法
2. 获取该股票最近 3 次分析记录
3. 分析预测准确性、观点演进、建议更新
4. **保存连续性分析报告**到中间数据目录（使用 `save-data` 命令）
5. 主 agent 使用 Read 工具读取 `intermediate/<股票名>/最新连续性.md` 获取结果

### 步骤 3: 自适应深度搜索

创建 subagent，要求 subagent 执行深度搜索任务收集新闻和热点信息。

**执行原则**（主 agent 操作）：
- 创建专门的搜索 subagent
- 在 subagent 的 prompt 中明确要求其执行深度搜索

**任务要求**（传递给 subagent 的 prompt 内容）：
1. subagent读取 `references/deep_search_guide.md` 了解详细执行方法
2. 优先使用 searxng-search-skill（本地部署，免费），遇到验证码切换到 brave-search
3. 按以下维度执行深度搜索：
   - 公司基本面（财务数据、经营指标、业务进展）
   - 行业生态（竞争格局、行业趋势、产业链）
   - 政策环境（行业政策、宏观影响、地区政策）
   - 实时动态（最近 1-2 周的新闻热点）
4. **保存搜索报告**到中间数据目录（使用 `save-data --type deep-search`）
5. 主 agent 使用 Read 工具读取 `intermediate/<股票名>/最新搜索.md` 获取搜索结果

### 步骤 4: 获取广发证券专业数据

创建 subagent，要求 subagent 调用 gf-finance skill 执行专业数据获取任务。

**执行原则**（主 agent 操作）：
- 创建专门的 subagent
- 在 subagent 的 prompt 中明确要求其调用 gf-finance skill 完成数据获取

**任务要求**（传递给 subagent 的 prompt 内容）：
1. subagent 读取 `references/gf_finance_guide.md` 了解执行指南
2. 调用 gf-finance skill 获取以下三维度数据：
   - **龙虎榜数据**：净买入/卖出、知名席位动向
   - **财务基本面**：PE/PB估值分位、ROE、现金流
   - **ETF 榜单**：相关ETF涨幅、资金流入
3. **保存广发数据汇总**到中间数据目录（使用 `save-data --type gf-summary`）
4. 主 agent 使用 Read 工具读取 `intermediate/<股票名>/最新广发.md` 获取专业数据

### 步骤 5: 整合中间数据

**主 agent 操作**：使用 Read 工具读取所有 subagent 保存的中间数据文件

```bash
# 读取历史连续性分析
Read intermediate/<股票名>/最新连续性.md

# 读取深度搜索报告
Read intermediate/<股票名>/最新搜索.md

# 读取广发证券数据
Read intermediate/<股票名>/最新广发.md
```

**数据整合要点**：
- ✅ 确保所有中间文件都已生成
- ✅ 提取各文件的关键信息
- ✅ 识别数据之间的关联和矛盾
- ✅ 准备好结构化数据供报告生成使用

### 步骤 6: 生成分析报告

按照三层分析框架生成 Markdown 报告：

**三层分析框架**：
1. **公司业务** - 核心业务状态、产品矩阵进展、财务表现
2. **行业趋势** - 行业周期位置、竞争格局变化、政策环境影响
3. **短期线索** - 正面信号、负面信号、催化剂事件

## 注意事项

### 执行流程相关
- **搜索迭代**：根据搜索结果质量调整搜索词，进行深度挖掘
- **数据验证**：如股价数据获取失败，在报告中标注"数据不可用"
- **报告保存**：股票分析完成后立即保存，避免丢失

### 工具使用相关
- **Skill 调用**：股价数据和广发数据通过创建 subagent 调用对应 SKILL 完成，主 agent 无需感知具体路径
- **数据验证**：如数据获取失败，在报告中标注"数据暂不可用"

### 数据分析相关
- **三层分析框架**：必须包含公司业务、行业趋势、短期线索三个层面
- **明确预测**：涨跌预测不可模棱两可，必须给出明确判断（⬆️/⬇️/➡️）
- **风险提示**：至少列出 3 项主要风险
- **新闻数量**：最新新闻摘要至少 5 条

### Subagent 管理
- **必须等待**：所有 subagent 必须等待任务完成并获取返回结果后再继续
- **中间数据保存**：每个 subagent 完成任务后，必须将结果保存到 `intermediate/<股票名>/` 目录
- **主 agent 读取**：主 agent 使用 Read 工具读取软链接（`最新XXX.md`）获取最新中间数据

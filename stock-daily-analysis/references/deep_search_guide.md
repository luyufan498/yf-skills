# 🔍 自适应深度搜索执行指南

## 概述

本指南定义了股票深度搜索 subagent 的执行方法和策略，旨在通过多轮迭代搜索和线索追踪，获取全面、深度的股票相关信息。

**核心理念**: 多轮迭代 + 线索追踪 + 全面覆盖

---

## 📋 搜索覆盖范围（四大维度）

### 1️⃣ 公司基本面
- **财务数据**: 营收、净利、毛利率、ROE、现金流
- **经营指标**: 销量、订单、产能利用率、库存水平
- **业务进展**: 新品发布、合作项目、扩张计划

### 2️⃣ 行业生态
- **竞争格局**: 主要对手动态、市场份额变化
- **行业趋势**: 增长/衰退周期、技术创新、消费变化
- **产业链**: 供应链稳定、上下游关系

### 3️⃣ 政策环境
- **行业政策**: 监管变化、补贴政策、准入门槛
- **宏观影响**: 财政政策、货币政策、贸易政策
- **地区政策**: 地方扶持、区域发展重点

### 4️⃣ 实时动态
- **最近 1-2 周**: 短期事件、临时公告
- **最近 3-5 天**: 新闻热点、突发消息
- **今日焦点**: 即时动态、论坛讨论

---

## 🔄 自适应搜索链条逻辑

### 初始搜索方向（覆盖四大维度）

```
第 1 轮 → 实时新闻聚焦（最近 1-2 周）
第 2 轮 → 公司财报与经营数据
第 3 轮 → 行业竞争与趋势
第 4 轮 → 政策与监管动态
```

### 自适应迭代（根据前期结果判断下一步）

```
每轮搜索后分析 → 提取关键线索
  ├─ 发现重大事件？→ 深挖相关背景与影响
  ├─ 竞争对手有动作？→ 搜索对手详情与对比
  ├─ 政策提及？→ 搜索政策全文与解读
  ├─ 行业术语？→ 搜索行业专业分析
  └─ 时间序列线索？→ 追溯历史或预测未来
```

### 终止条件（满足任一即可）
- 已覆盖四大维度且信息充分
- 连续 2 轮搜索未发现新关键信息
- 搜索轮数达到 8 轮（最多）

---

## 🎯 搜索关键原则

1. **线索驱动**: 从搜索结果中提取关键词、公司名、事件名、政策名，作为下一轮搜索词

2. **深度优先**: 发现重要线索时，优先深度挖掘而非广度扩展

3. **时效敏感**: 优先搜索最近时间区间的信息，实时性 > 历史性

4. **交叉验证**: 对同一信息（如财报数据）尝试不同搜索词验证

5. **信息溯源**: 追踪重要信息来源，必要时搜索源头

## ⚙️ 执行配置

### 搜索工具

本指南主要说明深度搜索的原则性方法和策略，具体搜索工具的使用请参考对应的 Skill 文档。

**可用搜索 Skill：**

1. **searxng-search（优先使用）** 🌟
   - 类型: 已安装的搜索 skill
   - 搜索工具: `scripts/searx-bash`
   - 搜索服务器: `http://192.168.100.2:38080`
   - 特点: 开源元搜索引擎，本地部署，免费使用，可聚合多个来源
   - 使用方法: 参考该 Skill 的 README.md 和脚本说明

2. **brave-search（备用方案）**
   - 工具: `web_search` (Claude Code 内置)
   - 搜索引擎: Brave Search API（provider: brave）
   - 特点: 高质量搜索结果，反爬虫能力强，API 稳定，付费服务
   - 已配置 API Key: ✅
   - 用途: 仅在 searxng 遇到验证码或访问限制时使用

3. **multi-search-engine（最后备用）**
   - 类型: 可选的搜索 skill
   - 特点: 支持多搜索引擎并行搜索，可绕过部分限制
   - 使用方法: 参考该 Skill 的 README.md 使用说明

**工具选择策略：**
- **优先使用**: `searxng-search-skill` - 本地部署，免费使用，可控性强
- **切换条件**: 如遇到验证码、访问受限等问题无法获取搜索结果时，切换到 `brave-search`
- **最后备用**: 如 brave-search 仍无法使用，尝试 `multi-search-engine`
- **切换说明**: 工具切换不影响本指南定义的搜索策略和迭代链条，仅替换底层搜索引擎

**说明**: 推荐优先使用 searxng-search-skill（本地部署，免费），仅在遇到验证码或访问限制时切换到 brave-search（付费 API）。multi-search-engine 作为最后的备用方案。本指南重点是搜索策略、迭代链条和覆盖维度的原则性说明。具体工具调用、参数配置等细节信息，请参考所选工具的官方文档。

### 执行步骤

**步骤一：初始化搜索环境**
- 选择搜索工具：优先使用 searxng-search-skill（本地部署，免费），遇到验证码或访问限制时切换到 brave-search（已配置 API Key）
- 参考 searxng-search-skill 的文档完成环境配置和工具初始化
- 确保搜索服务可用后开始执行

**步骤二：自适应搜索链条**
- 使用所选搜索工具执行搜索，每轮限制 15 条结果
- 按四大维度（实时动态、公司基本面、行业生态、政策环境）覆盖
- 根据每轮结果自主判断下一轮搜索方向
- 提取搜索结果中的关键线索（公司名、事件名、政策名）作为下一轮搜索词

**步骤三：深度挖掘**
- 发现重大事件？→ 深挖相关背景与影响
- 竞争对手有动作？→ 搜索对手详情与对比
- 政策提及？→ 搜索政策全文与解读
- 最多执行 8 轮搜索

**步骤四：返回结构化结果**
- 按四大维度整理搜索结果
- 记录搜索链条（每轮搜索词和关键发现）
- 标注信息缺失项

---

## ⏱️ 超时与耗时

- **超时设置**: `runTimeoutSeconds=600` (10 分钟) - 自适应链条可能需要 5-8 轮搜索，每轮需分析判断
- **预计耗时**: 3-8 分钟（自适应链条长度、信息密度）
- **时间不足会导致**: 搜索链条中断，信息不完整

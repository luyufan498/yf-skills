## 分析算法

### 1. 关键词提取

从用户输入中提取以下类型的关键词：

```
输入: "在IT环境测试寄存器配置时，发现当配置错误时系统没有正确处理异常"

提取结果:
- 环境: "IT"
- 模块: "寄存器"
- 功能: "配置"、"异常"
- 问题类型: "错误"
```

### 2. 匹配算法

对于每个字段，按以下顺序匹配：

```
步骤1: 精确匹配
  - 查找用户输入中是否包含精确关键词
  - 返回对应的推荐值

步骤2: 模糊匹配
  - 使用字符串相似度（Levenshtein距离）
  - 阈值设为 0.8（相似度 ≥ 80%）

步骤3: 上下文推断
  - 基于问题描述的上下文推断
  - 例如: 提及"配置错误" → ERROR_HAND

步骤4: 默认值
  - 使用字段的最常见值
  - 标记为"需用户确认"
```

### 3. 置信度计算

```
高置信度 (H):
  - 精确匹配关键词
  - 上下文明确指向

中置信度 (M):
  - 模糊匹配且相似度高
  - 上下文合理推断

低置信度 (L):
  - 多个可能值
  - 默认值推测
```

### 4. 冲突解决

当多个关键词匹配同一字段时：

```
优先级规则:
  1. 用户明确指定的值
  2. 高置信度匹配
  3. 功能相关性
  4. 默认值
```

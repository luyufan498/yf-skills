# 股票深度分析

执行系统化的股票深度分析流程，通过自适应搜索和多数据源整合，生成专业的股票分析报告。

## 核心流程

分析流程采用并行执行策略，通过多个专用 subagent 同时处理不同维度任务：

- [ ] 获取股价数据 - 实时股价、涨跌幅、K 线趋势 + 标准化股票信息
- [ ] 读取历史分析 - 建立分析连续性和趋势对比
- [ ] 自适应深度搜索 - 多轮迭代收集新闻和热点
- [ ] 获取专业数据 - 龙虎榜、财务基本面、ETF 榜单
- [ ] 整合中间数据 - 读取所有 subagent 保存的中间数据
- [ ] 获取模拟盘持仓 - 读取当前持仓状态和资金池信息
- [ ] 生成并保存分析报告 - 三层分析框架 + 模拟盘状态 + 结构化存储
- [ ] 输出执行摘要 - 完成状态和关键信息总结

**执行特点**：
- ⚡ **并行执行**：历史、搜索、广发数据三个 subagent 并行工作
- 🔗 **统一管理**：所有 subagent 由主 session 创建，确保完全隔离
- 📊 **多维整合**：历史、新闻、专业数据、股价全面融合


## 执行步骤

### 步骤 1: 获取股价数据

**主 agent 直接执行**：使用 ptrade CLI 工具获取实时股价、K 线数据以及标准化的股票名称和代码。

**执行原则**（主 agent 操作）：
- 主 agent 直接使用 ptrade 命令
- 用户输入的名称可能不准确，进行模糊匹配和验证，确保获取到正确的股票名称和代码，供后续步骤使用
- 获取标准化股票信息后，将其保存为全局变量或传递给后续步骤使用

**执行步骤**（主 agent 直接操作）：
1. 使用 ptrade search 命令搜索股票，获取标准化的股票名称和代码：
   ```bash
   ptrade search "<用户输入>" --limit 5
   ```
2. 根据搜索结果匹配用户意图，确定正确的股票名称和代码
3. 使用 `ptrade fetch-price <股票代码>` 获取实时股价数据：
   - 当前价、昨收、涨跌幅、成交量
4. 使用 `ptrade fetch-kline <股票代码> --type day --count 30` 获取 30 日日线趋势
   - **股票信息**：
     - 完整的股票名称（中文全称）
     - 正确的股票代码（如：600519.SH、000858.SZ）
     - 所属交易所信息（上交所/深交所）
   - **实时股价**：当前价、昨收、涨跌幅、成交量
   - **K 线数据**：30 日日线趋势
2. 如数据获取失败，标注"数据暂不可用，建议使用东方财富/同花顺查看"
3. 使用标准化股票名称(简体中文)和代码继续后续步骤

### 步骤 2: 读取历史分析

创建 subagent，要求 subagent 使用 ptrade CLI 工具读取历史分析报告并生成连续性分析。

**执行原则**（主 agent 操作）：
- 创建专门的 subagent
- 在 subagent 的 prompt 中明确要求其读取历史分析数据并保存连续性报告

**任务要求**（传递给 subagent 的 prompt 内容）：
1. subagent 读取 `references/analysis_history_guide.md` 了解执行方法
2. 使用 `ptrade analysis <股票名> --action read --count 5` 获取最近 5 次分析记录
3. 分析预测准确性、观点演进、建议更新
4. **保存连续性分析报告**到 temp-data 目录（使用 `ptrade temp-data --action save --category history-continuity` 命令）

### 步骤 3: 自适应深度搜索

创建 subagent，要求 subagent 执行深度搜索任务收集新闻和热点信息。

**执行原则**（主 agent 操作）：
- 创建专门的搜索 subagent
- 在 subagent 的 prompt 中明确要求其执行深度搜索

**任务要求**（传递给 subagent 的 prompt 内容）：
1. subagent 读取 `references/deep_search_guide.md` 了解详细执行方法
2. 优先使用 searxng-search-skill（本地部署，免费），遇到验证码切换到 brave-search
3. 获取市场新闻：使用 `ptrade fetch-news --source all --limit 30`
4. 按以下维度执行深度搜索：
   - 公司基本面（财务数据、经营指标、业务进展）
   - 行业生态（竞争格局、行业趋势、产业链）
   - 政策环境（行业政策、宏观影响、地区政策）
   - 实时动态（最近 1-2 周的新闻热点）
5. **保存搜索报告**到 temp-data 目录（使用 `ptrade temp-data --action save --category deep-search` 命令）

### 步骤 4: 获取广发证券专业数据

创建 subagent，要求 subagent 调用 gf-finance skill 执行专业数据获取任务。

**执行原则**（主 agent 操作）：
- 创建专门的 subagent
- 在 subagent 的 prompt 中明确要求其调用 gf-finance skill 完成数据获取

**任务要求**（传递给 subagent 的 prompt 内容）：
1. subagent 读取 `references/gf_finance_guide.md` 了解执行指南
2. 调用 gf-finance skill 获取以下三维度数据：
   - **龙虎榜数据**：净买入/卖出、知名席位动向
   - **财务基本面**：PE/PB估值分位、ROE、现金流
   - **ETF 榜单**：相关ETF涨幅、资金流入
3. **保存广发数据汇总**到 temp-data 目录（使用 `ptrade temp-data --action save --category gf-summary` 命令）

### 步骤 5: 获取模拟盘持仓

**主 agent 操作**：通过调用 paper-trading skill 获取当前模拟盘持仓状态和收益情况。

**执行步骤**：
1. 使用 Skill 工具调用 `paper-trading:SKILL.md`，读取该 skill 的使用文档，了解 ptrade 命令的使用方法

2. 尝试调用 `ptrade info <股票名>` 命令获取当前账户完整信息：
   - 如果命令成功，解析返回的资金池、持仓、收益信息
   - 如果返回"未找到账户"或类似错误，则执行步骤 3

3. 如果账户未初始化，自动初始化资金池（默认 500,000 元）：
   ```bash
   ptrade init "<股票名>" --capital 500000
   ```

4. 重新调用 `ptrade info <股票名>` 确认初始化成功并获取账户信息

5. 解析返回结果，提取以下关键信息：
   - **资金池状态**：
     - 总资金
     - 可用资金
     - 占用资金
     - 资金使用率
   - **当前持仓**：
     - 持仓数量
     - 持仓成本
     - 当前价格（如有）
   - **收益情况**：
     - 实现盈亏
     - 浮动盈亏
     - 总盈亏
     - 总收益率

6. 保存持仓和收益状态到变量，供报告生成使用

7. 如果初始化了资金池，在报告中标注"本次分析已创建模拟盘资金池"

**数据整合要点**：
- ✅ 持仓情况将影响投资建议的表述（如空仓可积极建仓，重仓则谨慎）
- ✅ 收益情况用于评估历史操作的有效性
- ✅ 记录当前持仓状态作为报告的"基准状态"
- ✅ 为后续异步执行交易行为提供基础数据

---

### 步骤 6: 整合中间数据

**主 agent 操作**：使用 ptrade temp-data 命令读取所有 subagent 保存的中间数据

```bash
# 读取历史连续性分析
ptrade temp-data "<股票名>" --action read --category history-continuity

# 读取深度搜索报告
ptrade temp-data "<股票名>" --action read --category deep-search

# 读取广发证券数据
ptrade temp-data "<股票名>" --action read --category gf-summary
```

**数据整合要点**：
- ✅ 确保所有中间数据都已生成并可以读取
- ✅ 提取各数据的关键信息
- ✅ 识别数据之间的关联和矛盾
- ✅ 准备好结构化数据供报告生成使用

### 步骤 7: 生成并保存分析报告

按照三层分析框架组织报告内容，**使用 ptrade CLI 工具保存**至规范化位置。

**三层分析框架**：
1. **公司业务** - 核心业务状态、产品矩阵进展、财务表现
2. **行业趋势** - 行业周期位置、竞争格局变化、政策环境影响
3. **短期线索** - 正面信号、负面信号、催化剂事件

**保存命令**：
```bash
# 步骤 1: 先保存到临时文件
cat > /tmp/{股票名}_analysis_${CLAUDE_SESSION_ID}.md << 'EOF'
<按三层分析框架组织的完整分析报告内容>
EOF

# 步骤 2: 使用 --file 参数归档
ptrade analysis "<股票名>" --action save --file /tmp/{股票名}_analysis_${CLAUDE_SESSION_ID}.md
```

**保存位置**: `stocks_analysis/<股票名>/<股票名>-YYYY-MM-DD-HHMM.md`
- 该命令会自动创建带时间戳的文件名
- 自动创建 Soft链接 `最新分析.md` 指向最新分析

## 注意事项

### 执行流程相关
- **搜索迭代**：根据搜索结果质量调整搜索词，进行深度挖掘
- **数据验证**：如股价数据获取失败，在报告中标注"数据不可用"
- **报告保存**：股票分析完成后立即保存，避免丢失

### 工具使用相关
- **Skill 调用**：股价数据、广发数据、模拟盘交易系统通过创建 subagent 调用对应 SKILL 完成，主 agent 无需感知具体路径
  - paper-trading skill：使用 Skill 工具调用 `paper-trading:SKILL.md` 自行读取文档了解 ptrade 命令使用方法
- **数据验证**：如数据获取失败，在报告中标注"数据暂不可用"

### 数据分析相关
- **三层分析框架**：必须包含公司业务、行业趋势、短期线索三个层面
- **明确预测**：涨跌预测不可模棱两可，必须给出明确判断（⬆️/⬇️/➡️）
- **风险提示**：至少列出 3 项主要风险
- **新闻数量**：最新新闻摘要至少 5 条

### Subagent 管理
- **必须等待**：所有 subagent 必须等待任务完成并获取返回结果后再继续
- **中间数据保存**：每个 subagent 完成任务后，必须将结果保存到 ptrade 的 `temp-data/<股票名>/` 目录（通过 `ptrade temp-data` 命令）
- **主 agent 读取**：主 agent 使用 `ptrade temp-data --action read --category <类别>` 命令读取临时数据，替代文件路径读取，保持工具使用统一性
- **主 agent 不读取参考文档**：`references/` 目录下的所有 `.md` 文件（如 `analysis_history_guide.md`、`deep_search_guide.md`、`gf_finance_guide.md`）是给 subagent 执行时阅读的。主 agent 无需也不应自己去读这些文件，只需在创建 subagent 的 prompt 中指明让 subagent 读取对应文档即可

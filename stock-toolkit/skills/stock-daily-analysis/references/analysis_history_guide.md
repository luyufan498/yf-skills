# 📜 历史分析连续性指南

## 🎯 核心目标

确保股票分析的**连续性**和**一致性**，通过历史预测对比提供回测数据，追踪观点演进，避免分析矛盾。

---

## 💡 为什么需要历史分析连续性

### 问题场景
- ❌ 今天说"看多"，明天说"看空"，没有连贯逻辑
- ❌ 上次预测"目标价 60 元"，这次完全忽略
- ❌ 建议演进不清晰：昨天说"持有"，今天说"加仓"，没有理由
- ❌ 风险提示遗忘：上次提到的风险这次完全不提

### 解决方案
- ✅ 读取历史分析，确保观点连贯
- ✅ 对比历史预测与实际走势，评估准确性
- ✅ 追踪观点演进，说明变化原因
- ✅ 持续关注已识别的风险

---

## 🎯 执行步骤

### 步骤 1: 列出历史分析记录

首先列出该股票的历史分析报告，了解分析频次和时间分布。

```bash
ptrade analysis 股票名称 --action list --limit 15
```

参数说明：
- `--action list`: 列出操作
- `--limit 15`: 列出最近 15 次分析报告（可根据需要调整）
- 命令返回格式：`股票名称-YYYY-MM-DD-HHMM.md`，包含完整文件路径

### 步骤 2: 智能选择分析报告

根据列出的历史报告，智能选择具有代表性的报告进行读取：

**选择策略**：
1. **短期分析**（最近 1-3 天）：选择最近 3-5 份报告
   - 了解最新的观点变化
   - 追踪短期预测验证情况
   - 识别最新的关注焦点

2. **中期分析**（一周左右）：选择 5-7 天前的 2-3 份报告
   - 评估中期预测准确性
   - 对比观点演进趋势
   - 验证关键判断的延续性

**智能选择示例**：
```bash
# 假设今天是 2026-04-13，列出的报告如下：
# - 2026-04-13-1400.md (今天下午) ✓ 选择
# - 2026-04-13-0942.md (今天上午) ✓ 选择
# - 2026-04-12-2216.md (昨天) ✓ 选择
# - 2026-04-10-1402.md (3天前) ✓ 选择
# - 2026-04-10-1127.md (3天前) - 同一天多份，选择一份即可
# - 2026-04-09-1536.md (4天前) - 中期报告
# - 2026-04-08-0946.md (5天前) - 中期报告
# - 2026-04-07-0946.md (6天前) ✓ 选择
# - 2026-04-06-0947.md (7天前) ✓ 选择

# 最终选择 5-7 份报告进行读取
```

**注意事项**：
- 如果同一天有多份报告（如测试报告），优先选择时间最新或标记为正式分析的报告
- 如果历史报告少于 5 份，则全部读取
- 如该股票从未分析过，命令会返回"暂无历史分析记录"，此时跳过连续性分析步骤

### 步骤 3: 读取选定的历史报告

使用 `--action read` 命令读取智能选择的历史报告：

```bash
# 方式一：使用 --count 参数读取最近的报告（短期分析）
ptrade analysis 股票名称 --action read --count 3

# 方式二：使用 --file 参数读取指定的历史报告文件
ptrade analysis 股票名称 --action read --file /path/to/report.md
```

**推荐组合读取方式**：
```bash
# 读取最近的 3 份报告（短期）
ptrade analysis 股票名称 --action read --count 3

# 然后读取 5-7 天前的报告（中期）
# 使用 --file 参数指定具体文件路径（从列表中选择）
ptrade analysis 股票名称 --action read --file <中期报告路径>
```

### 步骤 4: 分析预测准确性

对比历史预测与实际走势：
- ✅ 预测方向准确性（看多/看空）
- ✅ 目标价位达成率
- ✅ 建议操作的有效性
- ✅ 关键观点验证
- ✅ 短期 vs 中期预测准确率对比

### 步骤 5: 识别观点演进

追踪历史分析的变化趋势：
- **观点变化**: 看空 → 看多 → 看空
- **建议演进**: 观望 → 持有 → 加仓 → 减仓
- **风险追踪**: 持续关注的风险点
- **时间维度**: 短期 vs 中期观点演进对比

### 步骤 5: 保存到中间数据目录

请参考下列命令将分析报告保存。

💾 保存方式：使用 ptrade CLI 工具的临时数据存储功能
📂 保存位置：ptrade 工作的 workspace/temp-data 目录（通过 STOCK_ANALYSIS_WORKSPACE 环境变量配置）

```bash
# 1. 先将连续性分析报告保存到临时文件
cat > /tmp/{股票名称}_history_continuity_${CLAUDE_SESSION_ID}.md << 'EOF'
{连续性分析报告内容}
EOF

# 2. 使用 --file 参数归档
ptrade temp-data "股票名称" \
  --action save \
  --category history-continuity \
  --file /tmp/{股票名称}_history_continuity_${CLAUDE_SESSION_ID}.md
```

⚠️ 严格禁止使用 write 或 Edit 工具直接操作文件，必须通过 ptrade temp-data 命令保存数据
**重要**: 保存成功后，其他 agent 会自动读取和合并这份报告。

## 📋 注意事项

### 数据目录说明

- **历史分析记录**: 保存在 `stocks_analysis/` 目录（股票名称作为目录）
- **中间数据**: 保存在 `temp-data/` 目录（股票名称作为目录，通过 ptrade 的工作空间配置）
- **分析文件命名**: `<股票名>-YYYY-MM-DD-HHMM.md`
- **首次分析**: 如从未分析过该股票，返回"暂无历史分析记录"

### 环境变量配置

确保已配置 `STOCK_ANALYSIS_WORKSPACE` 环境变量（项目级配置）：

```json
{
  "env": {
    "STOCK_ANALYSIS_WORKSPACE": "/path/to/workspace-stock_agent"
  }
}
```

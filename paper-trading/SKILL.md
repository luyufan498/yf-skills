# 模拟盘交易系统

## 概览

模拟盘交易系统是一个用于测试投资策略和练习股票交易的虚拟交易平台。支持A股和港股交易，每个股票有独立的资金池管理，提供完整的交易流程追踪和收益计算功能。

## 核心功能

### 资金池管理
- **独立资金池**：每只股票享有独立的资金池系统
- **实时状态**：随时查询可用资金、占用资金和使用率
- **灵活配置**：可自定义初始资金金额

### 交易操作
- **买入操作**：支持按股数和按金额两种买入方式
- **卖出操作**：支持部分卖出和全仓卖出
- **价格获取**：自动获取实时股价进行交易
- **成本计算**：采用先进先出(FIFO)法则计算成本

### 数据追踪
- **持仓报告**：详细显示持仓数量、成本、市值和浮动盈亏
- **操作历史**：记录所有交易流水和时间戳
- **收益分析**：统计实现盈亏和浮动盈亏，计算收益率

## 快速开始

### 初始化资金池

为指定股票创建独立的资金池：

```bash
python3 <skill目录>/scripts/paper_trading.py init "股票名称" --capital 100000
```

**参数说明**：
- `股票名称`：必需，如 "赛力斯"、"中科曙光"
- `--capital` / `-c`：必需，初始资金金额
- `--code`：可选，股票代码（如 sh600000），不提供时自动查询
- `--force` / `-f`：可选，强制删除现有资金池并重新初始化

**示例**：
```bash
# 正常初始化
python3 <skill目录>/scripts/paper_trading.py init "赛力斯" --capital 50000

# 强制重新初始化
python3 <skill目录>/scripts/paper_trading.py init "赛力斯" --capital 100000 --force
```

### 查询资金池状态

```bash
python3 <skill目录>/scripts/paper_trading.py pool "股票名称"
```

### 买入股票

```bash
# 按股数买入
python3 <skill目录>/scripts/paper_trading.py buy "股票名称" --qty 100

# 按金额买入
python3 <skill目录>/scripts/paper_trading.py buy "股票名称" --amount 10000

# 添加备注
python3 <skill目录>/scripts/paper_trading.py buy "股票名称" --qty 200 --note "初始建仓"
```

### 卖出股票

```bash
# 按股数卖出
python3 <skill目录>/scripts/paper_trading.py sell "股票名称" --qty 50

# 全仓卖出
python3 <skill目录>/scripts/paper_trading.py sell "股票名称" --all

# 添加备注
python3 <skill目录>/scripts/paper_trading.py sell "股票名称" --qty 100 --note "部分止盈"
```

### 查看持仓报告

```bash
python3 <skill目录>/scripts/paper_trading.py holdings "股票名称"
```

### 查看操作历史

```bash
python3 <skill目录>/scripts/paper_trading.py operations "股票名称"
```

### 生成报告

```bash
# 持仓报告
python3 <skill目录>/scripts/paper_trading.py report "股票名称" --type holdings

# 收益报告
python3 <skill目录>/scripts/paper_trading.py report "股票名称" --type profit

# 操作报告
python3 <skill目录>/scripts/paper_trading.py report "股票名称" --type history
```

## 交易原则与策略

### 核心交易纪律

1. **优先基于分析报告操作**
   - 所有交易决策应基于每日分析报告的投资建议
   - 不得在无分析报告支持的情况下进行交易
   - 交易备注中应关联对应的分析报告

2. **严格执行止损止盈**
   - 买入前设定明确的止损价和止盈价
   - 达到止损线及时止损
   - 达到止盈线分批止盈

3. **仓位控制原则**
   - 单只股票初始建仓不超过总资金的 30%
   - 空仓时首次建仓仓位控制在总资金的 10-20%
   - 重仓股（占比 > 50%）应谨慎加仓

4. **分批买卖原则**
   - 大额资金建议分 2-3 批次逐步建仓
   - 上涨趋势中突破关键位可加仓
   - 下跌趋势中跌破支撑位减仓或止损

5. **避免频繁交易**
   - 每日交易次数不超过 2 次（同一股票）
   - 避免日内短线操作
   - 相同方向交易间隔至少 1 个交易日

### 禁止行为

1. **禁止盲目追涨杀跌**
   - 不因短期快速上涨盲目追高
   - 不因短期快速下跌恐慌杀跌
   - 以分析报告为依据，避免情绪化操作

2. **禁止频繁操作**
   - 一日内多次买卖同一股票
   - 频繁改变交易方向
   - 无交易逻辑的随机操作

3. **禁止满仓梭哈**
   - 单只股票持仓不超过总资金 50%
   - 保留一定现金应对机会
   - 避免一次建仓占用全部资金

4. **禁止摊平死扛**
   - 亏损达到止损线严格执行止损
   - 不在亏损时增加仓位摊低成本
   - 及时止损，保存本金

### 交易决策流程

```
1. 生成每日分析报告
   ↓
2. 评估投资建议（买入/卖出/持有）
   ↓
3. 检查当前持仓状态
   ↓
4. 评估资金使用率
   ↓
5. 制定具体交易计划
   - 数量/金额
   - 分批策略
   - 止损/止盈位
   ↓
6. 执行交易（添加详细备注）
   ↓
7. 记录并总结
```

## 数据存储

交易数据存储在 `intermediate/<股票名>/模拟买卖/` 目录下：

```
intermediate/
└── 中科曙光/
    └── 模拟买卖/
        ├── holdings.json     # 当前持仓、资金池状态
        └── operations.json   # 历史操作记录
```

### holdings.json
存储当前持仓和资金池状态，包含：
- 股票基本信息
- 资金池状态（总资金、可用资金、占用资金）
- 持仓列表（每次买入/卖出的仓位记录）

### operations.json
存储所有历史操作记录，包含：
- 初始化操作
- 买入操作（价格、数量、金额、备注）
- 卖出操作（价格、数量、金额、成本、盈亏、备注）

## 常见问题

### 问题 1：找不到股票代码

**解决方法**：
1. 检查股票名称是否正确
2. 手动指定股票代码：`--code sh600000`
3. 使用其他查询工具确认股票代码

### 问题 2：资金不足

**解决方法**：
1. 减少买入数量或金额
2. 先卖出部分持仓释放资金
3. 使用 `--force` 重新初始化更大资金池

### 问题 3：无法获取实时价格

**解决方法**：
1. 检查网络连接
2. 确认股票代码格式正确（sh/sz/hk 开头）
3. 尝试其他股票代码测试

## 注意事项

1. **美股支持**：本系统仅支持 A股和港股，不支持美股代码
2. **资金验证**：买入操作前会严格检查资金池余额
3. **持股验证**：卖出操作前会验证持仓数量
4. **实时价格**：每次交易操作都会自动获取最新价格
5. **数据持久化**：所有操作实时保存到 JSON 文件
6. **强制初始化**：使用 `--force` 会永久删除现有数据
